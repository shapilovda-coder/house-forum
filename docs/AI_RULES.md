# Правила работы с AI (Kimi Claw)

## Жёсткие ограничения

### 1. Ветки
- ✅ Работать **только в `ai-dev`**
- ❌ **Никогда не пушить в `main`**
- ❌ **Никогда не пушить в `master`**

### 2. Pull Requests
- ✅ **Всегда создавать PR** перед merge
- ✅ PR должен пройти проверку (Vercel preview)
- ❌ Никаких прямых пушей в main

### 3. Git безопасность
- ❌ **Никогда не делать force push**
- ❌ **Никогда не переписывать историю** (rebase -i, amend)
- ❌ **Никогда не удалять ветки** (без согласования)

### 4. Изменения
- ❌ **Никаких массовых переименований** без согласования
- ❌ **Никаких изменений CI/CD** без согласования
- ❌ **Никаких изменений next.config** без согласования
- ❌ **Не удалять файлы без подтверждения** (показать список)

### 5. Архитектура
- ❌ **Не делать архитектурные изменения** без отдельной задачи
- ❌ **Не менять структуру проекта** без согласования
- ✅ Сначала стабилизация, потом рефакторинг

### 6. Данные
- ❌ **Не коммитить data/backup/** (в .gitignore)
- ❌ **Не коммитить .env**
- ❌ **Не коммитить API ключи**

## Workflow

### Ежедневная работа
1. `git checkout main`
2. `git pull origin main`
3. `git checkout -b ai-dev` (или использовать существующую)
4. Править код
5. `git add -A`
6. `git commit -m "описание"`
7. `git push origin ai-dev`
8. Создать PR на GitHub
9. Ждать approve и merge

### Если правило нарушается
**ОСТАНОВИТЬСЯ И СПРОСИТЬ**

Никогда не продолжать, если не уверен.
