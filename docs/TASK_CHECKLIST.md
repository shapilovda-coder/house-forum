# Чеклист перед изменением (TASK CHECKLIST)

## Для каждой задачи — прочитать и проверить:

### 1. Какой тип задачи?
- [ ] Фронтенд (UI/UX)
- [ ] Бэкенд (API/данные)
- [ ] Контент (текст/SEO)
- [ ] Инфраструктура (деплой/домены)

### 2. Что меняю?
```
Файл: ________________
Строки: ______________
Что меняется: ________
```

### 3. Проверка на конфликты:

#### Если меняю HomePage.tsx:
- [ ] Hero banner на месте? (строка с `/hero-banner.svg`)
- [ ] Плитки категорий есть? (grid с фото)
- [ ] StekloRoll/Artalico первые? (sort)
- [ ] Адрес в карточках? (company.address)
- [ ] Телефон в карточках? (company.phone)

#### Если меняю стили:
- [ ] Цвета бренда? (#3B82F6, #F97316)
- [ ] Не сломал адаптив? (mobile/desktop)

#### Если меняю данные:
- [ ] StekloRoll и Artalico в БД?
- [ ] Поле address заполнено?
- [ ] Supabase ключ рабочий?

#### Если меняю роуты:
- [ ] `[[...slug]]` не сломан?
- [ ] `generateStaticParams` работает?
- [ ] 404 страница есть?

### 4. После изменения:
- [ ] `npm run build` проходит?
- [ ] Ошибок в консоли нет?
- [ ] Визуально проверил?
- [ ] Задеплоил?
- [ ] Проверил на проде?

### 5. Обновил документацию?
- [ ] PROJECT_CONTEXT.md (если правила изменились)
- [ ] SESSION_LOG.md (что сделал)
