# Решение проблемы контекста

## Причина

Я не читаю файлы памяти перед сессией. Деплой ≠ потеря контекста. Потеря контекста = я не посмотрел PROJECT_BIBLE.

## Решение: Жёсткий чеклист

### Перед КАЖДЫМ сообщением:

```
1. Прочитать PROJECT_BIBLE.md
2. Прочитать memory/TODAY.md (или последний)
3. Только потом отвечать
```

### Если не прочитал — не отвечаю!

## Структура для пользователя

### Файл: PROJECT_CONTEXT.txt (короткий!)

```
=== СТРОЙСЕЙЛС — КОНТЕКСТ ===

БЮДЖЕТ: $0 (бесплатно)
СТЭК: Next.js + Supabase + Vercel

ЧТО НЕЛЬЗЯ:

- Менять карточки без согласования
- Деплоить без staging

ТЕКУЩИЕ ЗАДАЧИ:
- [ ] Офисные перегородки — найти компании
- [ ] СПб поставщики — по 10 на категорию
- [ ] Деплой — ждём разблокировки Vercel

- Supabase: zzellrqkamskeftyprkv.supabase.co
- Vercel: заблокирован до завтра

```

### Этот файл — 1 минута на чтение

## Моя обязанность

**Перед каждым ответом:**
1. Открыть PROJECT_CONTEXT.txt
2. Проверить, что помню
3. Если не помню — перечитать
4. Отвечать с учётом контекста

## Проверка

Пользователь может спросить:
- "Прочитал PROJECT_BIBLE?"
- "Помнишь про бюджет?"
- "Какая у нас SEO-стратегия?"

Если не отвечаю — значит не прочитал.

## Что изменится

| Было | Стало |
|------|-------|
| Спрашиваю повторно | Знаю ответ из файла |
| Теряю контекст | Читаю перед сессией |
| Делаю ошибки | Следую PROJECT_BIBLE |

---

**Решение:** Жёсткий чеклист перед каждым ответом.
