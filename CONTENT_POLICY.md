# StroySales Content Integrity Policy

## Что произошло (инцидент 2026-03-20)

### Причина потери контента
1. **Коммит `88f7749`** — полные SEO-статьи (1500-1800 слов) в `public/blog/`
2. **Коммит `5228e8a`** — статьи перемещены в `app/blog/` как TSX-компоненты
3. **Результат** — контент сокращен до ~300-400 слов (только основные тезисы)
4. **Последствия** — потеря SEO-ценности, 404 на старые ссылки (временно)

### Почему это произошло
- Рефакторинг структуры ради единообразия с Next.js App Router
- Потеря контента при конвертации HTML → TSX
- Отсутствие проверки объема контента перед коммитом

---

## Правила для статей (Content Policy)

### 1. Объем контента
- **Минимум:** 1000 слов для SEO-статьи
- **Оптимум:** 1500-2000 слов
- **Проверка:** `wc -w article.html` перед коммитом

### 2. Локация статей
**Решено:** Использовать `public/blog/` (статические HTML)
- ✅ Быстрая загрузка (pre-rendered)
- ✅ Нет зависимости от Next.js runtime
- ✅ Простое копирование/резервное копирование
- ❌ Не использовать `app/blog/` для длинных статей

### 3. Структура статьи (обязательно)
```
- H1: Основной заголовок с ключевым словом
- Введение (150-200 слов): проблема + обещание решения
- H2: Что такое [продукт] (200 слов)
- H2/H3: Виды/типы (400-500 слов)
- Таблица цен (обязательно)
- H2: Плюсы и минусы (300 слов)
- H2: Как выбрать (200 слов)
- CTA-блок со ссылкой на StekloRoll
- Выводы (100 слов)
```

### 4. Чек-лист перед публикацией
- [ ] Объем 1000+ слов (`wc -w`)
- [ ] Есть таблица цен
- [ ] Есть CTA на StekloRoll
- [ ] Уникальный title, description
- [ ] Проверка ссылок (нет 404)
- [ ] Хлебные крошки работают
- [ ] Фото/изображения (опционально)

### 5. Запрещено
- ❌ Удалять статьи без архива
- ❌ Сокращать существующие статьи без согласования
- ❌ Переносить HTML → TSX (теряется контент)
- ❌ Менять URL без редиректа

## Процесс публикации новой статьи

1. Создать файл `public/blog/[slug]/index.html`
2. Проверить объем: `wc -w index.html`
3. Добавить в `public/blog/index.html` (список всех статей)
4. Обновить `app/blog/page.tsx` (Next.js блог-страница)
5. Проверить build: `npm run build`
6. Проверить ссылки локально
7. Git commit → push
8. Проверить на проде через 5 минут

## Контроль качества

### Команды для проверки
```bash
# Проверить объем всех статей
for f in public/blog/*/index.html; do echo "$f: $(wc -w < $f) words"; done

# Проверить битые ссылки
find public/blog -name "index.html" -exec grep -o 'href="[^"]*"' {} \; | sort | uniq

# Проверить наличие CTA
grep -r "stekloroll" public/blog/*/index.html
```
